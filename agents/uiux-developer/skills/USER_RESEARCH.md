# User Research

## Purpose
Gather and analyze user needs, behaviors, and pain points to inform design decisions.

## Serves Goals
- Usability
- User satisfaction

## Inputs
- `knowledge/AUDIENCE.md` (existing personas)
- `journal/entries/` (user feedback, bug reports, feature requests)
- Product-owner requirements from `outputs/`
- `MEMORY.md` (past research findings)

## Process
1. Read current audience profiles
2. Identify research question (what do we need to learn?)
3. Analyze existing user feedback from journal entries
4. Conduct competitive analysis (how do Slack, Discord, Teams handle this?)
5. Identify user pain points specific to the feature area
6. Map user journey for the feature (steps, emotions, friction points)
7. Document findings and design implications
8. Propose updates to AUDIENCE.md if new insights found

## Outputs
- `outputs/YYYY-MM-DD_ux-research_[topic].md`
- Journal entry with key findings

## Quality Bar
- Research question is clearly stated
- ≥2 competitive products analyzed
- User journey includes friction points
- Findings directly inform design recommendations
- Proposals for AUDIENCE.md updates if applicable
